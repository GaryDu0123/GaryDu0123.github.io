Three CSS style sheets

According to the position of CSS style writing (or the way it is introduced), CSS style sheets can be divided into three categories:

● Inline style sheet (inline style)

Inline style sheet (inline style sheet) is to set the CSS style in the style attribute inside the element tag. Suitable for modifying simple styles

1 <div style="color: red; font-size: 12px;">hi</div>

● style is actually the attribute of the label

● In the middle of the double quotes, the wording must conform to the CSS specification

● Can control the current label setting style

● Because the writing is cumbersome and does not reflect the idea of separating structure and style, it is not recommended to use it in large quantities. Only when adding a single style to the current element, you can consider using it

● But high priority is called inline quoting

● Internal style sheet (embedded)

1 <style>

2 div {

3     color: red;

4     font-size: 12px;

5 }

6 </style>

● The <style> tag can theoretically be placed anywhere in the HTML document, but it is generally placed in the <head> tag of the document

● In this way, you can easily control the current element style settings in the entire page

● The code structure is clear, but the structure and style are not completely separated

● Use the internal style sheet to set CSS, which is usually called embedded import. This method is commonly used when we practice.

● External style sheet (linked)

● The actual development is all external style sheets. It is suitable for the situation with more styles. The core is: the style is written in the CSS file separately, and then the CSS file is introduced into the HTML page for use

● The introduction of an external style sheet is divided into two steps:

● Create a new style file with a suffix of .css, and put all CSS codes into this file.

● In the HTML page, use the <link> tag to import this file.

1 <link rel="stylesheet" href="css/path">

The horizontal layout of the elements:

margin-left + border-left + padding-left + width + padding-right + border-right + margin-right =The width of the content area of the entire parent element

The above equation must be satisfied. If the result of the addition makes the equation invalid, it is called a transition constraint, and the equation will be adjusted automatically.

● If none of these seven values is auto, the browser will automatically adjust the margin-right value to satisfy the equation

● If a value is auto, it will be automatically adjusted to the value of auto to make the equation true

● If you set a width and a margin to auto, the width will be adjusted to the maximum, and the margin set to auto will automatically be 0

● If the two outer margins are set to auto and the width is fixed, the margins will be set to the same value

● So we often use this feature to make an element horizontally centered in its parent element

Relative positioning

● When the position attribute of the element is set to relative, the relative positioning of the element is turned on

● Characteristics of relative positioning

● After the relative positioning of the element is turned on, if the offset is not set, there will be no change to the element user

● Relative positioning is based on the position of the element in the document flow for positioning

● Relative positioning will improve the level of elements

● Relative positioning will not deviate from the document flow

● Relative positioning does not change the nature of the element. Block or block, inline or inline

Absolute positioning

● When the position attribute value of the element is set to absolute, the absolute positioning of the element is turned on-the characteristics of absolute positioning:

● After the absolute positioning is turned on, the position of the element will not change if the offset is not set

● After turning on absolute positioning, the element will be separated from the document flow

● Absolute positioning will change the nature of the element, the line will become a block, and the width and height of the block will be stretched by the content

● Absolute positioning will raise the element one level

● Absolutely positioned elements are positioned relative to their containing block

● The containing block is the nearest ancestor block element of the current element

● In the case of opening absolute positioning

● The containing block is the nearest ancestor element that has been positioned

● If all the ancestor elements have not opened positioning, then positioning is relative to the root element
